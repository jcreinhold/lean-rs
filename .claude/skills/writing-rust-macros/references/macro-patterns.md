# Rust Macro Patterns Reference

## Fragment Specifier Reference

| Specifier | Matches | Follow-set restrictions |
| --- | --- | --- |
| `expr` | Any expression (2024+: includes `_` and `const`) | `=>`, `,`, `;` |
| `expr_2021` | Expression without `_` or `const {}` at top | `=>`, `,`, `;` |
| `ty` | Any type | `=>`, `,`, `=`, `\|`, `;`, `:`, `>`, `>>`, `[`, `{`, `as`, `where`, `block` frag |
| `path` | Type path (`std::fmt::Debug`) | Same as `ty` |
| `pat` | Pattern (2021+: includes top-level `\|`) | `=>`, `,`, `=`, `if`, `in` |
| `pat_param` | Pattern without top-level `\|` | `=>`, `,`, `=`, `\|`, `if`, `in` |
| `ident` | Identifier or keyword (not `_`) | Any token |
| `tt` | Single token tree (token or delimited group) | Any token |
| `stmt` | Statement (no trailing `;`) | `=>`, `,`, `;` |
| `block` | `{ ... }` block | Any token |
| `item` | Function, struct, impl, etc. | Any token |
| `meta` | Attribute contents | Any token |
| `vis` | Visibility qualifier (possibly empty) | `,`, identifier (not raw `priv`), any type-start token |
| `lifetime` | `'a`, `'static`, etc. | Any token |
| `literal` | `-?` + literal value | Any token |

**Edition variants:** `expr_2021` for pre-2024 `expr` behavior. `pat_param` for pre-2021 `pat` behavior. The relevant
edition is the one at the `macro_rules!` definition.

**Opacity warning:** Capturing with anything except `ident`, `lifetime`, and `tt` renders the captured AST **opaque** —
it cannot be further matched in subsequent macro invocations. Use `tt` if cross-macro re-parsing is needed.

## Repetition Operators

```rust
$( $x:expr ),*    // Zero or more, comma-separated
$( $x:expr ),+    // One or more, comma-separated
$( $x:expr )?     // Zero or one (no separator with ?)
$(,)?              // Optional trailing comma
```

Multiple metavariables in the same repetition must bind the same number of fragments.

## Architecture: Parse → Emit

Every non-trivial declarative macro should separate parsing from code generation:

```rust
macro_rules! my_macro {
    // Public entry point
    ( $($input:tt)* ) => {
        my_macro!(@parse [] $($input)*);
    };

    // @parse: accumulate normalized entries
    (@parse [$($acc:tt)*] some_pattern, $($rest:tt)*) => {
        my_macro!(@parse [$($acc)* [normalized_entry]] $($rest)*);
    };

    // @parse: terminal — hand off to emit
    (@parse [$($entries:tt)+]) => {
        my_macro!(@emit $($entries)+);
    };

    // @emit: generate final code
    (@emit $($entry:tt)+) => { /* code generation */ };
}
```

## Named Patterns

### 1. Stamp-Out (simplest)

```rust
macro_rules! impl_display {
    ($($ty:ty),*) => {
        $( impl MyDisplay for $ty { /* ... */ } )*
    };
}
impl_display!(u8, u16, u32, u64, i8, i16, i32, i64);
```

### 2. Internal Rules (`@label`)

Separate parsing from generation within one macro:

```rust
macro_rules! m {
    ($($input:tt)*) => { m!(@internal $($input)*); };
    (@internal $x:ident) => { /* single ident */ };
    (@internal $x:ident, $($rest:tt)*) => { /* list */ };
}
```

### 3. TT Munching (recursive token processing)

```rust
macro_rules! count {
    (@count [] -> $n:expr) => { $n };
    (@count [$_head:tt $($tail:tt)*] -> $n:expr) => {
        count!(@count [$($tail)*] -> ($n + 1))
    };
    ($($tokens:tt)*) => { count!(@count [$($tokens)*] -> 0) };
}
```

**Warning:** O(n²) in token count. Keep recursion depth < 20.

### 4. Push-Down Accumulation

```rust
macro_rules! reverse {
    (@reverse [] -> [$($acc:tt)*]) => { ($($acc)*) };
    (@reverse [$head:tt $($tail:tt)*] -> [$($acc:tt)*]) => {
        reverse!(@reverse [$($tail)*] -> [$head $($acc)*])
    };
    ($($tokens:tt)*) => { reverse!(@reverse [$($tokens)*] -> []) };
}
```

### 5. Callback/Replay (inter-crate)

A proc macro generates a `macro_rules!` replay macro; downstream crates invoke it with their own callback:

```rust
// Generated by proc macro in crate A:
macro_rules! __replay_metadata {
    ($callback:ident) => {
        $callback! { entry1 { field1: Type1 }; entry2 { field2: Type2 }; }
    };
}

// Consumed in crate B:
crate_a::__replay_metadata!(my_code_generator);
```

### 6. Conditional Struct Generation

```rust
macro_rules! define_expr {
    (has_effect: false) => { struct Expr { kind: Kind, ty: Type } };
    (has_effect: true)  => { struct Expr { kind: Kind, ty: Type, eff: Effect } };
}
```

## Known Limitations

| Limitation | Workaround |
| --- | --- |
| Cannot concatenate identifiers | `paste` crate: `paste! { [<visit_ $name>] }` |
| Cannot convert case | `paste` crate: `$name:snake`, `$name:camel` |
| Cannot index tuple fields (\_0, \_1) | Hardcoded counter arms or proc macro |
| Cannot introspect types/fields | Proc macro with `syn` |
| Cannot generate distinct names | Proc macro for indexing |

## Procedural Macros

### Three Types

| Type | Annotation | Input | Use Case |
| --- | --- | --- | --- |
| Derive | `#[proc_macro_derive(Name, attributes(...))]` | Struct/enum definition | Generate impls from types |
| Attribute | `#[proc_macro_attribute]` | Args + annotated item | Transform any item |
| Function-like | `#[proc_macro]` | Token stream | Custom syntax, DSLs |

### Structure: Parse + Transform

```rust
#[proc_macro_derive(MyDerive, attributes(my_attr))]
pub fn derive_my_thing(input: TokenStream) -> TokenStream {
    let input = parse_macro_input!(input as DeriveInput);
    match derive_my_thing_impl(&input) {
        Ok(tokens) => tokens.into(),
        Err(err) => err.to_compile_error().into(),
    }
}
```

### Handling Generics

```rust
let (impl_generics, ty_generics, where_clause) = input.generics.split_for_impl();
Ok(quote! {
    impl #impl_generics MyTrait for #name #ty_generics #where_clause { ... }
})
```

Never manually reconstruct generic parameters. `split_for_impl()` handles all edge cases.

### Attribute Parsing (syn 2.x)

```rust
attr.parse_nested_meta(|meta| {
    if meta.path.is_ident("skip") { skip = true; Ok(()) }
    else if meta.path.is_ident("rename") {
        rename = Some(meta.value()?.parse::<LitStr>()?.value());
        Ok(())
    } else { Err(meta.error("unknown attribute")) }
})?;
```

### Error Handling

- Use `syn::Error`, not `panic!`
- Accumulate with `syn::Error::combine` for multiple issues
- Point to spans: `syn::Error::new_spanned(tokens, message)`

### Compile-Fail Testing

```rust
#[test]
fn compile_fail_tests() {
    let t = trybuild::TestCases::new();
    t.compile_fail("tests/ui/*.rs");
}
```

## Hygiene and Scoping

- **Local variables/labels**: definition-site context (hygienic)
- **Functions/types/modules**: invocation-site context (unhygienic)
- **`$crate::`** in `#[macro_export]` macros for cross-crate paths
- **Textual scope** (default) vs **path-based scope** (`#[macro_export]`)
